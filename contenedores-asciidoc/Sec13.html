Agregar persistencia al deployment

Creaci√≥n del recurso PersistenVolumeClaim
cat <<EOF > pvc.yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: mysql-data
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
EOF

kubectl apply -f pvc.yaml
Modificaci√≥n de la aplicaci√≥n para que soporte persistencia
cat <<EOF > deployment-persistent.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  creationTimestamp: null
  labels:
    app: mysql-deployment
  name: mysql-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: mysql-deployment
  strategy:
    type: Recreate
  template:
    metadata:
      creationTimestamp: null
      labels:
        app: mysql-deployment
    spec:
      volumes:
        - name: mysql-volume
          persistentVolumeClaim:
            claimName: mysql-data
      containers:
      - image: quay.io/sclorg/mysql-80-c9s
        name: mysql
        env:
          - name: MYSQL_USER
            value: myuser
          - name: MYSQL_PASSWORD
            value: mypassword
          - name: MYSQL_DATABASE
            value: mydb
        ports:
        - containerPort: 3306
        volumeMounts:
          - name: mysql-volume
            mountPath: /var/lib/mysql/data
EOF

kubectl apply -f deployment-persistent.yaml
Verificar el volume persistente
kubectl exec deployment/mysql-deployment -- df -h /var/lib/mysql/data

Aprovisionamiento est√°tico

Eliminar el deployment y PVC anteriores
kubectl delete -f deployment-persistent.yaml
kubectl delete -f pvc.yaml
En lb, generar el directorio para el PersistentVolume
mkdir -p /shared/static
chmod 777 /shared/static
Generar el recurso PersistentVolume en formato YAML. Desde m1
cat <<EOF > pv.yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: mysql-static
spec:
  nfs:
    server: nfs.s1.example.com
    path: /shared/static
  accessModes:
    - ReadWriteMany
    - ReadWriteOnce
    - ReadOnlyMany
  capacity:
    storage: 10Gi
EOF
Generar el PersistentVolumeClaim est√°tico
cat <<EOF > pvc-static.yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: mysql-data
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
  storageClassName: "" (1)
EOF

  1. Este atributo indica que estamos tratando de utilizar un PV Est√°tico

Aplicar el PVC y el Deployment como anteriormente
kubectl apply -f pvc-static.yaml
